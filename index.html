01Einführung

Was ist iSCSI?

iSCSI steht für Internet Small Computer System Interface – also für eine Methode, mit der ein Computer Speicher über ein normales IP-Netzwerk so nutzt, als wäre die Festplatte lokal eingebaut.

🔌

Die Idee in einem Satz

Ein iSCSI-Target stellt Speicher im Netzwerk bereit, und dein Rechner bindet ihn so ein, als läge der Datenträger direkt im Gerät.

🧵

Die einfache Analogie

Stell dir ein sehr langes USB-Kabel vor, das durchs Netzwerk läuft. Am anderen Ende hängt eine Festplatte, die du ganz normal benutzen kannst.

Warum gibt es das überhaupt?

Weil zentraler Speicher praktisch ist. Statt in jedem Server neue Platten einzubauen, kann man Speicher sauber bündeln, flexibel verteilen und über vorhandenes Ethernet nutzen.

02Grundlagen

Die 5 Begriffe, die du kennen musst

Wenn du diese Begriffe sicher kennst, ist das Grundprinzip von iSCSI fast schon verstanden.

Initiator
Der Client. Also der Rechner, der Speicherplatz haben oder nutzen möchte.
Target
Das Zielsystem. Ein Server oder NAS, das die eigentlichen Speicherbereiche bereitstellt.
LUN
Eine logische Einheit, die sich für den Client wie eine eigene Festplatte verhält.
IQN
Der eindeutige Name eines iSCSI-Geräts, zum Beispiel iqn.2024-01.de.firma:speicher1.
Port 3260
Der typische Standard-Port, über den iSCSI-Verbindungen aufgebaut werden.

04Anwendung

Wofür wird iSCSI verwendet?

Vor allem dort, wo zentraler, flexibel zuteilbarer Speicher gebraucht wird.

🖥️

Virtualisierung

Virtuelle Maschinen können zentral gespeichert und leichter verwaltet werden.

🏢

SAN

Unternehmen bauen Speichernetze auf, um mehrere Systeme mit Blockspeicher zu versorgen.

💽

Backups

Große Datensicherungen landen auf einem zentralen Speicherziel.

🗃️

Datenbanken

Viele Datenbanksysteme profitieren von Speicher, der sich wie ein lokaler Blockdatenträger verhält.

💰

Kostenvorteil

iSCSI ist oft günstiger als spezielle SAN-Technologien mit eigener Hardware.

📈

Skalierung

Speicher lässt sich je nach Bedarf erweitern, ohne jeden Rechner intern umbauen zu müssen.

05Abwägung

Vorteile und mögliche Nachteile

iSCSI ist sehr praktisch, bringt aber wie jede Netzwerktechnik auch ein paar Bedingungen mit.

✅ Vorteile

  • +Günstig: Nutzt vorhandenes Ethernet statt Spezial-Hardware.
  • +Flexibel: Speicher kann zentral verteilt und erweitert werden.
  • +Block-Speicher: Für viele Systeme wirkt der Datenträger lokal.
  • +Vertraut: Admins arbeiten mit bekannten IP- und Ethernet-Konzepten.
  • +Skalierbar: Gut für Virtualisierung, Backups und zentrale Serverlandschaften.

⚠️ Mögliche Nachteile

  • –Netzabhängig: Langsame oder ausgelastete Netze bremsen direkt den Speicherzugriff.
  • –Latenz: Etwas träger als eine direkt eingebaute Platte.
  • –Sicherheit: Ohne Schutzmaßnahmen ist Zugriff im Netz ein Risiko.
  • –Planungsaufwand: Für gute Performance braucht man oft getrennte Netze oder VLANs.
  • –CPU-Last: Das Kapseln und Verarbeiten der Pakete kostet Rechenleistung.

💡 Merksatz: Die meisten Probleme von iSCSI entstehen nicht durch das Protokoll selbst, sondern durch ein schlecht geplantes oder überlastetes Netzwerk.

06Praxis

iSCSI in der Praxis: ein Mini-Projekt

So könnte ein typischer Ablauf in einer kleinen Firma aussehen.

📖 Das Szenario

Ein Speicher-Server hat noch viel freien Platz. Ein anderer Server braucht mehr Kapazität, kann intern aber nicht erweitert werden. Also wird Speicher per iSCSI im Netzwerk bereitgestellt und eingebunden.

1

Auf dem Target: Speicher bereitstellen

Auf dem Speicher-Server wird eine virtuelle Festplatte angelegt und über ein iSCSI-Target freigegeben.

# 20 GB große virtuelle Festplatte anlegen
/backstores/fileio create disk1 /storage/disk1.img 20G

# Ein Target mit eindeutigem Namen erstellen
/iscsi create iqn.2024-01.de.muster:speicher1

# Die LUN dem Target zuordnen
/iscsi/iqn.20.../tpg1/luns create /backstores/fileio/disk1
2

Zugriff absichern

Damit nur berechtigte Systeme verbinden dürfen, wird häufig CHAP für Anmeldung und Zugriffsschutz verwendet.

3

Auf dem Initiator: Verbinden

Der Client sucht nach verfügbaren Targets und meldet sich beim richtigen Speicherziel an.

# Targets suchen
iscsiadm -m discovery -t st -p 192.168.1.10

# Mit dem Target verbinden
iscsiadm -m node --login
4

Die neue Festplatte benutzen

Danach erscheint auf dem Initiator ein zusätzlicher Datenträger, der formatiert, partitioniert und wie eine normale Festplatte genutzt werden kann.

Für Programme sieht dieser Speicher oft so aus, als wäre er direkt lokal im System eingebaut – genau das macht iSCSI so nützlich.

Zusammenfassung in Kürze

iSCSI stellt Blockspeicher über ein IP-Netzwerk bereit und verbindet damit die Flexibilität zentraler Speicherung mit der Arbeitsweise einer lokalen Festplatte.

Was? Ein Protokoll, das Speicherbefehle über TCP/IP überträgt.
Wie? Ein Initiator verbindet sich mit einem Target und nutzt eine LUN.
Wofür? Für Virtualisierung, SAN, Backups und zentrale Serverumgebungen.
Stärke? Günstig, flexibel und mit vorhandener Netzwerktechnik nutzbar.
Schwachpunkt? Die Qualität des Netzwerks entscheidet stark über Leistung und Stabilität.

🚀 Warum iSCSI weiterhin wichtig bleibt

Je schneller und sauberer moderne Netzwerke werden, desto sinnvoller wird zentraler Speicher über IP. Gerade in virtualisierten Umgebungen, kleinen Rechenzentren und flexiblen Infrastrukturen bleibt iSCSI deshalb ein relevantes Werkzeug.
